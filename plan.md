# @chzkyli/template 优化与升级计划

该项目是一个功能完备的 Deno 项目脚手架。目前已完成核心重构与功能增强，后续将侧重于生态扩展与工程深度优化。

## 1. 核心功能增强 (Core Features)
- [x] **参数化运行**：支持命令行参数（如 `--name`, `--template`, `--git`），允许跳过交互式提示。
- [ ] **更多模板支持**：增加常用的 Deno 框架模板，如 **Hono**, **Fresh**, **Oak** 等。
- [ ] **许可证选择**：允许用户从常见许可证（MIT, Apache-2.0, GPL-3.0）中选择。
- [ ] **自定义远程模板**：支持通过 URL 下载并应用自定义模板包。

## 2. 模板引擎升级 (Template Engine)
- [ ] **引入模板引擎**：引入 `Handlebars` 或类似的轻量级引擎，取代简单的 `replaceAll` 字符串替换，以支持条件渲染和循环逻辑。
- [ ] **动态变量注入**：支持在模板中注入更多变量（如 `author`, `year`, `description`）。

## 3. 用户体验优化 (DX/UX)
- [ ] **配置持久化**：支持 `.templaterc` 配置文件，记录用户偏好（如默认作者、是否开启 Biome 等）。
- [x] **更新检查**：在 CLI 启动时检查 JSR 新版本并提示。
- [ ] **更好的错误处理**：针对网络波动、文件权限等异常提供更具指导性的错误信息。

## 4. 工程质量保证 (Quality Assurance)
- [x] **模块化重构**：将逻辑拆分为 `cli`, `generator`, `update` 等独立模块，提升可维护性。
- [x] **单元测试**：建立了基础测试套件，验证核心逻辑的正确性。
- [ ] **集成测试**：增加针对 `generator` 的测试，验证生成的文件结构和内容是否符合预期。
- [ ] **代码覆盖率**：配置测试覆盖率报告，确保核心逻辑 100% 覆盖。

## 5. 持续交付与自动化 (CI/CD)
- [ ] **自动化发布流**：配置 GitHub Actions，在合并到主分支或打 Tag 时自动执行 Lint、Test 并发布到 JSR。
- [ ] **CHANGELOG 自动生成**：使用工具自动管理版本变更日志。
